Project Proposal: Dynamic Lineup-Dependent Defensive Scheme Optimizer 
By Gabriele Lisci (gl618) and Victor Olivo (vmo32)

1. Problem Definition & Strategic Aspects Our project addresses the limitations of static defensive game-planning in the NBA. Currently, coaching staffs often rely on overall team averages or "gut feeling" to assign defensive coverages (e.g., Switch-All, Drop, 2-3 Zone), failing to account for the specific 5-man lineups currently on the floor or immediate situational directives (e.g., "Protect the Paint"). The problem is a lack of real-time, data-driven systems capable of instantly analyzing the specific five players on the court to recommend the mathematically optimal defensive coverage.
Strategically, this project bridges core course concepts by combining advanced database management with predictive modeling. We will build an end-to-end pipeline that aggregates granular, individual play-type data using relational SQL databases and feeds it into a machine learning classification and regression engine to output situational game-plan recommendations.
2. Novelty, Importance, and Prior Work The importance of this project lies in its ability to optimize real-time decision-making in high-stakes environments. Exposing a weak defender or misaligning a coverage is heavily punished in the modern, highly-spaced NBA; optimizing these decisions can yield a measurable Expected Points Saved (EPS).
Prior Work & Gap Identification: Existing sports analytics literature heavily covers pre-game outcome predictions (e.g., E.g., Predicting NBA Win Percentage, or standard Plus-Minus models). Additionally, while proprietary models exist within NBA front offices, public-facing analytics (like ESPN's BPI or 538's RAPTOR) focus on evaluating a player's past value rather than prescribing future schematic tactics. The literature lacks a dynamic, open-source recommendation engine that integrates live coaching directives (constraints) with real-time lineup data. Our project fills this gap by shifting from descriptive analytics to prescriptive, explainable AI.

Course Relation to Lectures and Discussions: The project will show off key concepts from this course by demonstrating applications based on the teachings of the end-to-end data science pipeline, database setup, and predictive modeling. 4. Data Description and Accessibility
Data Source & Accessibility: The data is 100% free, publicly accessible, and legally available via the open-source nba_api Python client, which queries the NBA's official stats database (stats.nba.com).
Format: The raw data will be fetched in JSON format and processed into tabular Pandas DataFrames before being loaded into a structured SQL database.
Features: The primary features include individual defensive Points Per Possession (PPP) allowed across distinct Synergy play-types (Isolation, Pick-and-Roll Ball Handler, Pick-and-Roll Roll Man, Spot-Up). The target variable is the overall DEF_RATING (Defensive Points allowed per 100 possessions) of specific 5-man lineup combinations.
5. Methodology & Implementation Plan Our implementation plan follows a structured, end-to-end data science lifecycle:
Database Setup: We will design a relational SQL database utilizing JOINs and GROUP BYs to map individual Player_Defense metrics to Lineup_Metrics and Historical_Matchups.
Model Selection: We will employ Tree-Based Models, specifically XGBoost (Extreme Gradient Boosting) or a Random Forest Regressor. These algorithms are selected because they excel at capturing non-linear relationships and feature interactions (e.g., how a center's poor perimeter defense interacts with a guard's high screen-navigation stats) better than standard linear regression.
Explainability: We will integrate SHAP (SHapley Additive exPlanations) values to translate the model's mathematical output into human-readable insights (e.g., explaining why the model rejected a Switch-All scheme), satisfying the need for actionable intelligence.
6. Evaluation Metrics and Success Criteria
Predictive Accuracy: The model's baseline predictions will be evaluated using Mean Absolute Error (MAE) and Root Mean Squared Error (RMSE). We will compare our model's predicted Defensive Rating for a given scheme against the actual historical play-by-play outcomes in a holdout test dataset (the final 20 games of the season).
Recommendation Success (Business Metric): The ultimate success metric is "Expected Points Saved." We will backtest our recommendations to verify if the model's #1 suggested scheme statistically resulted in a lower opponent Points Per Possession than the alternative schemes.
